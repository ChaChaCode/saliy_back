# API админ-панели: Подписчики рассылки

**Базовый URL:** `/api/admin/newsletter`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

Управление подписчиками маркетинговой рассылки. Сами подписки создаются публичным эндпоинтом [POST /api/newsletter/subscribe](../shop/newsletter.md) (форма на сайте). В админке можно смотреть кто подписан, фильтровать активных/отписавшихся и удалять записи.

---

## 1. Статистика подписчиков

**GET** `/api/admin/newsletter/stats`

| Поле ответа | Описание |
|-------------|----------|
| `total` | Всего подписчиков |
| `active` | Активных подписчиков |
| `unsubscribed` | Отписавшихся |

Используйте `active` чтобы показывать «N человек подписано». `unsubscribed` — кто когда-то подписался и отписался.

---

## 2. Список подписчиков

**GET** `/api/admin/newsletter`

| Параметр | Тип | Описание |
|----------|-----|----------|
| `isActive` | true \| false | Только активные / только отписавшиеся |
| `search` | string | Поиск по email (case-insensitive) |
| `page` | number | По умолчанию 1 |
| `limit` | number | По умолчанию 50 |

Ответ: объект с полями `subscribers` (массив) и `pagination`. Каждый подписчик содержит:

| Поле | Описание |
|------|----------|
| `id` | UUID записи |
| `email` | Email подписчика |
| `isActive` | Активна ли подписка |
| `source` | Откуда подписка: `footer` / `popup` / `checkout` / null (для аналитики) |
| `acceptedTerms` | Согласие с условиями |
| `unsubscribeToken` | Секретный токен; ссылка на отписку: `https://saliystudio.com/api/newsletter/unsubscribe/{token}` |
| `userId` | id зарегистрированного пользователя или `null` (гость) |
| `subscribedAt` | Дата подписки |
| `unsubscribedAt` | Дата отписки или `null` |

---

## 3. Удалить подписчика

**DELETE** `/api/admin/newsletter/:id`

Физическое удаление записи. Для мягкой отписки (с сохранением истории) подписчик переходит по unsubscribe-ссылке из письма — тогда `isActive` станет `false`, но запись останется.

Ответ: `success: true`.

Когда использовать: DELETE — спам, ошибочный ввод, дубликат, запрос на удаление по GDPR; мягкая отписка (через клиентскую ссылку) — обычная отписка, история сохраняется.

---

## Связь с email-кампаниями

Подписчики используются в [API кампаний](./campaigns.md) как отдельная аудитория. При создании рассылки выберите `targetType: "NEWSLETTER"` — письмо уйдёт только активным подписчикам с автоматически добавленным unsubscribe-футером.

Для рассылок не подписчикам (например, всем пользователям с заказами) unsubscribe-футер не добавляется — это транзакционные уведомления, не маркетинг. Использование `targetType: "ALL"` для маркетинга без согласия (`acceptedTerms`) — серая зона по ФЗ 152.

## Типичный UX в админке

1. Дашборд — показывать `stats.active` как один из ключевых KPI («аудитория рассылки»).
2. Страница подписчиков — таблица: email, источник, дата подписки, активен/отписан. Фильтры по `isActive` и `search`.
3. Кампании — при создании выбрать `NEWSLETTER` чтобы попасть только в подписчиков, готовых к маркетингу.
4. Удаление — кнопка «удалить» с подтверждением (физическое удаление по GDPR-запросам).
