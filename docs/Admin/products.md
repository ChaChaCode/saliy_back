# API управления товарами (Admin)

Административный API для управления товарами.

**Базовый URL:** `/api/admin/products`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

---

## Типы данных

### CardStatus (статус карточки товара)

| Значение | Описание |
|----------|----------|
| `NONE` | Без метки |
| `NEW` | Новинка |
| `SALE` | Распродажа |
| `SOLD_OUT` | Распродано |
| `PRE_ORDER` | Предзаказ |
| `COMING_SOON` | Скоро в продаже |

### GenderType (пол)

| Значение | Описание |
|----------|----------|
| `male` | Мужской |
| `female` | Женский |
| `unisex` | Унисекс |

### CategoryType (тип категории)

| Значение | Описание |
|----------|----------|
| `TOP` | Верхняя одежда |
| `BOTTOM` | Нижняя одежда |
| `ACCESSORIES` | Аксессуары |
| `SPORT` | Спортивная одежда |
| `OTHER` | Другое |

### Объект Product

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | ID товара |
| `name` | string | Название товара |
| `slug` | string | URL-friendly идентификатор |
| `description` | string | Описание товара |
| `cardStatus` | CardStatus | Статус карточки |
| `gender` | GenderType | Для какого пола |
| `color` | string | Цвет товара |
| `weight` | number | Вес в граммах |
| `price` | number | Цена в рублях |
| `discount` | number | Скидка в процентах (0-100) |
| `images` | Array&lt;ImageObject&gt; | Массив изображений |
| `stock` | object | Остатки по размерам, например `{"S": 10, "M": 5}` |
| `isActive` | boolean | Активен ли товар |
| `viewCount` | number | Количество просмотров |
| `salesCount` | number | Количество продаж |
| `categories` | Array&lt;Category&gt; | Массив категорий |
| `createdAt` | string | Дата создания |
| `updatedAt` | string | Дата обновления |

### Объект ImageObject

| Поле | Тип | Описание |
|------|-----|----------|
| `url` | string | URL/S3-ключ изображения |
| `isPreview` | boolean | Является ли превью |
| `previewOrder` | number | Порядок превью (1 — основное, 2 — hover, 999 — обычное) |

## Эндпоинты

### Создать товар

**POST** `/api/admin/products`

Content-Type: `multipart/form-data`. Поддерживает загрузку изображений. **Первое загруженное изображение автоматически становится превью** (`isPreview: true`, `previewOrder: 1`).

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `name` | string | Да | Название товара |
| `slug` | string | Да | Уникальный слаг (URL) |
| `description` | string | Нет | Описание |
| `price` | number | Да | Цена в рублях (>= 0) |
| `discount` | number | Нет | Скидка 0-100 |
| `cardStatus` | enum | Нет | NONE / NEW / SALE / SOLD_OUT / PRE_ORDER / COMING_SOON |
| `gender` | enum | Нет | male / female / unisex |
| `color` | string | Нет | Цвет |
| `weight` | number | Нет | Вес в граммах (>= 0) |
| `stock` | JSON | Нет | Остатки по размерам, например `{"S": 10, "M": 5, "L": 0}` |
| `isActive` | boolean | Нет | По умолчанию `true` |
| `categoryIds` | number[] | Нет | ID категорий для привязки (JSON-массив или CSV) |
| `images[]` | file[] | Нет | До 10 изображений (jpg/png/webp, до 10MB каждое) |

Ответ: созданный товар со связями категорий.

Ошибки:

| Код | Описание |
|-----|----------|
| 400 | slug уже используется |
| 400 | неверный формат файла (только jpg/png/webp) |

---

### Получить список товаров

**GET** `/api/admin/products`

Список всех товаров с фильтрацией и пагинацией.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `page` | number | 1 | Номер страницы |
| `limit` | number | 20 | Товаров на странице |
| `search` | string | - | Поиск по названию, описанию, slug |
| `category` | string | - | Фильтр по slug категории |
| `gender` | string | - | Фильтр по полу (male, female, unisex) |
| `cardStatus` | string | - | Фильтр по статусу |
| `isActive` | boolean | - | Фильтр по активности |

Ответ: объект с полями `products` (массив объектов Product) и `pagination` (`page`, `limit`, `total`, `totalPages`).

---

### Получить товар по ID

**GET** `/api/admin/products/:id`

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID товара |

Ответ: объект Product. Ошибки: `404` — товар не найден, `400` — неверный ID.

---

### Получить enums для товаров

**GET** `/api/admin/products/enums/all`

Возвращает доступные значения enums и список категорий:

| Поле ответа | Тип | Описание |
|-------------|-----|----------|
| `genders` | string[] | male, female, unisex |
| `cardStatuses` | string[] | NONE, NEW, SALE, SOLD_OUT, PRE_ORDER, COMING_SOON |
| `categoryTypes` | string[] | TOP, BOTTOM, ACCESSORIES, SPORT, OTHER |
| `categories` | Array&lt;Category&gt; | Список категорий (id, name, slug, type) |

---

### Товары с низким остатком

**GET** `/api/admin/products/low-stock`

Товары, у которых остаток по каким-либо размерам не превышает порог. Полезно для отслеживания дозакупки.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `threshold` | number | 5 | Порог: возвращаются размеры с остатком ≤ threshold |

Поля ответа:

| Поле | Тип | Описание |
|------|-----|----------|
| `threshold` | number | Использованный порог |
| `count` | number | Количество найденных товаров |
| `products` | array | Товары с полями `id`, `name`, `slug`, `price`, `discount`, `imageUrl`, `lowSizes` (только размеры с остатком ≤ threshold), `totalStock` (суммарный остаток по всем размерам) |

### Обновить товар

**PATCH** `/api/admin/products/:id`

Content-Type: `multipart/form-data`. Поддерживает загрузку новых фотографий.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID товара |

Поля (все опциональные):

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Название |
| `slug` | string | Уникальный слаг (URL) |
| `description` | string | Описание |
| `cardStatus` | CardStatus | Статус карточки |
| `gender` | GenderType | Пол |
| `color` | string | Цвет |
| `weight` | number | Вес в граммах |
| `price` | number | Цена в рублях |
| `discount` | number | Скидка (0-100) |
| `stock` | object | Остатки, например `{"S": 10, "M": 5}` |
| `isActive` | boolean | Активность |
| `categoryIds` | number[] | ID категорий (старые связи заменяются) |
| `images[]` | file[] | Новые фотографии (до 10 файлов) |

Ответ: обновлённый товар. При передаче `categoryIds` старые связи с категориями удаляются и создаются новые.

---

### Удалить изображение товара

**PATCH** `/api/admin/products/:id/delete-image`

Content-Type: `application/json`. Удаляет конкретное изображение из S3 и из массива `images`.

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `imageUrl` | string | Да | Полный URL ИЛИ голый S3-ключ (`products/21/foo.jpg`). Бэк нормализует строку перед поиском |

Ответ: сообщение об успехе и обновлённый товар.

Ошибки:

| Код | Описание |
|-----|----------|
| 400 | Image not found in product — такого изображения у товара нет |
| 404 | Товар не существует |

---

### Добавить фото к товару

**POST** `/api/admin/products/:id/images`

Content-Type: `multipart/form-data`. Добавляет новые фото к существующему товару (без замены остальных). Новые фото добавляются как **не превью** (`previewOrder = 999`); назначить превью — через `set-previews`.

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `images[]` | file[] | Да | До 10 файлов (jpg/png/webp, до 10MB каждое) |

Ответ: сообщение о количестве добавленных фото и обновлённый товар.

---

### Назначить превью-фото

**PATCH** `/api/admin/products/:id/set-previews`

Content-Type: `application/json`. Назначает до 2 превью-фото товара.

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `primary` | string | Да | Основное фото (`previewOrder = 1`). Полный URL или S3-ключ |
| `hover` | string \| null | Нет | Фото при наведении (`previewOrder = 2`). `null` — оставить только одно превью |

Остальные фото товара автоматически теряют статус превью (`previewOrder = 999`).

Ошибки:

| Код | Описание |
|-----|----------|
| 400 | primary is required — не передан основной превью |
| 400 | primary и hover не могут быть одним и тем же изображением |
| 400 | primary (или hover) изображение не найдено у товара |
| 404 | Товар не существует |

---

### Удалить товар

**DELETE** `/api/admin/products/:id`

Логика удаления:
- Если у товара **нет** связанных заказов — товар физически удаляется из БД, файлы изображений удаляются. Каскадно удаляются связи с категориями и записи в корзинах.
- Если у товара **есть** связанные заказы — товар деактивируется (`isActive = false`), чтобы не нарушить историю заказов.

Ответ при физическом удалении: `success: true`, `deleted: true`, сообщение.
Ответ при деактивации: `success: true`, `deleted: false`, сообщение и объект товара.

Ошибки: `404` — товар не найден, `400` — неверный ID.

---

## Коды ошибок

| Код | Описание |
|-----|----------|
| 400 | Bad Request — некорректные данные |
| 401 | Unauthorized — отсутствует/недействителен токен |
| 403 | Forbidden — недостаточно прав |
| 404 | Not Found — товар не найден |
| 413 | Payload Too Large — файл слишком большой (лимит 10MB) |
| 500 | Internal Server Error — ошибка сервера |

---

## Примечания

1. **Загрузка изображений:** до 10 файлов за раз, до 10MB каждый, форматы JPG/JPEG/PNG/WEBP.
2. **Остатки товара (stock):** формат `{"размер": количество}`, например `{"S": 10, "M": 5, "L": 0}`.
3. **Размерная таблица** отдельным полем не хранится — загружайте фото таблицы как обычное изображение через `images[]`.
4. **Авторизация:** все эндпоинты требуют `AdminGuard`. Запросы авторизуются через httpOnly cookie `adminToken` (заголовок `Authorization` также принимается для обратной совместимости).
