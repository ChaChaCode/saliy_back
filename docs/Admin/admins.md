# API админ-панели: Управление администраторами

**Базовый URL:** `/api/admin/admins`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

---

## Роли

| Роль | Назначение |
|------|-----------|
| `SUPER_ADMIN` | Полный доступ ко всему |
| `ADMIN` | Управление заказами, товарами, пользователями |
| `MODERATOR` | Просмотр данных |

---

## 1. Список админов

**GET** `/api/admin/admins`

Ответ — массив администраторов с полями: `id`, `telegramId`, `name`, `role`, `isActive`, `createdAt`, `updatedAt`.

---

## 2. Получить админа по ID

**GET** `/api/admin/admins/:id`

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | string | UUID администратора |

---

## 3. Создать админа

**POST** `/api/admin/admins`

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `telegramId` | string | Да | Telegram User ID |
| `name` | string | Да | Имя администратора |
| `role` | AdminRole | Нет | SUPER_ADMIN / ADMIN / MODERATOR |

Ошибки: `400` — админ с таким Telegram ID уже существует.

---

## 4. Обновить админа

**PATCH** `/api/admin/admins/:id`

Все поля опциональны.

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Имя |
| `role` | AdminRole | SUPER_ADMIN / ADMIN / MODERATOR |
| `isActive` | boolean | Активность |

Деактивация (`isActive: false`) — альтернатива удалению: такой админ не сможет войти, но сохранится в БД.

---

## 5. Удалить админа

**DELETE** `/api/admin/admins/:id`

Ограничения: нельзя удалить самого себя. Рекомендуется использовать деактивацию через `PATCH`.

Ответ: `success: true`.
