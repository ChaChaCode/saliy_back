# API управления баннерами (Admin)

Административный API для управления баннерами главной страницы.

**Базовый URL:** `/api/admin/banners`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

---

## Типы данных

### Объект Banner

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | UUID баннера |
| `title` | string | Название баннера (для админки) |
| `description` | string \| null | Описание |
| `desktopImageUrl` | string | URL изображения для десктопа |
| `mobileImageUrl` | string | URL изображения для мобильной версии |
| `link` | string \| null | Ссылка при клике |
| `order` | number | Порядок отображения (0 = первый) |
| `isActive` | boolean | Активен ли баннер |
| `createdAt` | string | Дата создания |
| `updatedAt` | string | Дата обновления |

---

## Эндпоинты

### Получить список баннеров

**GET** `/api/admin/banners`

Список всех баннеров (включая неактивные), отсортированных по порядку отображения. Ответ — массив объектов Banner.

---

### Получить баннер по ID

**GET** `/api/admin/banners/:id`

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | string | UUID баннера |

Ответ: объект Banner. Ошибки: `404` — баннер не найден.

---

### Создать баннер

**POST** `/api/admin/banners`

Content-Type: `multipart/form-data`. Оба изображения обязательны.

| Поле | Тип | Обязательное | По умолчанию | Описание |
|------|-----|--------------|--------------|----------|
| `title` | string | Да | - | Название баннера |
| `desktopImage` | file | Да | - | Изображение для десктопа (jpg/png/webp, до 5MB) |
| `mobileImage` | file | Да | - | Изображение для мобильных (jpg/png/webp, до 5MB) |
| `description` | string | Нет | - | Описание |
| `link` | string | Нет | - | Ссылка при клике |
| `order` | number | Нет | `0` | Порядок отображения |
| `isActive` | boolean | Нет | `true` | Активность |

Ответ: созданный объект Banner. Ошибка `400` — не переданы оба изображения.

---

### Обновить баннер

**PATCH** `/api/admin/banners/:id`

Content-Type: `multipart/form-data`.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | string | UUID баннера |

Поля (все опциональные):

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | string | Название |
| `description` | string | Описание |
| `link` | string | Ссылка |
| `order` | number | Порядок отображения |
| `isActive` | boolean | Активность |
| `desktopImage` | file | Новое изображение для десктопа (замена) |
| `mobileImage` | file | Новое изображение для мобильных (замена) |

Ответ: обновлённый объект Banner. При замене изображений старые файлы удаляются автоматически.

---

### Удалить баннер

**DELETE** `/api/admin/banners/:id`

Удаляет баннер; файлы изображений удаляются автоматически. Операция необратима.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | string | UUID баннера |

Ответ: сообщение об успешном удалении.

---

## Коды ошибок

| Код | Описание |
|-----|----------|
| 400 | Bad Request — некорректные данные или отсутствуют обязательные поля/файлы |
| 401 | Unauthorized — отсутствует/недействителен токен |
| 403 | Forbidden — недостаточно прав |
| 404 | Not Found — баннер не найден |
| 413 | Payload Too Large — файл слишком большой (лимит 5MB) |
| 500 | Internal Server Error — ошибка сервера |

---

## Примечания

1. **Обязательные изображения при создании:** при создании нужны оба изображения (desktop и mobile); при обновлении можно заменить только одно.
2. **Загрузка изображений:** до 5MB на файл, форматы JPG/JPEG/PNG/WEBP; при обновлении старые файлы удаляются автоматически.
3. **Удаление баннера:** файлы изображений удаляются автоматически, операция необратима.
4. **Порядок отображения:** баннеры сортируются по полю `order` (0 = первый); можно использовать любые числа.
5. **Значения по умолчанию:** `order` — `0`, `isActive` — `true`, `id` генерируется автоматически (UUID).
6. **Авторизация:** все эндпоинты требуют `AdminGuard`. Запросы авторизуются через httpOnly cookie `adminToken` (заголовок `Authorization` также принимается для обратной совместимости).
7. **Формат имён файлов:** desktop — `desktop-banner-{id}-{timestamp}.{ext}`, mobile — `mobile-banner-{id}-{timestamp}.{ext}`.
8. **Рекомендуемые размеры:** desktop 1920x600px, mobile 768x768px.
