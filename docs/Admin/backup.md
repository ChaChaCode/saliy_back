# Бэкапы базы данных

Система резервного копирования PostgreSQL: автоматически по расписанию, по HTTP-запросу администратора и по команде в Telegram. Каждый бэкап — сжатый дамп всей базы, который отправляется в отдельный Telegram-канал и сохраняется локально на сервере.

---

## Что делает бэкап

- Снимает дамп всей базы (`pg_dump`), сжимает в gzip.
- Сохраняет локальную копию на сервере (каталог `backups/` в корне приложения).
- Отправляет файл в Telegram-канал дампов (переменная `TELEGRAM_CHANNEL_ID_DUMP`).
- Ротирует локальные копии — удаляет старше 14 дней.
- При любой ошибке (упал `pg_dump`, пустой дамп, не отправился файл) шлёт текстовый алерт в тот же канал.

Ограничение Telegram на размер файла бота — около 50 МБ. Если дамп больше, файл остаётся локально на сервере, а в канал приходит предупреждение с путём к файлу.

---

## Способы запуска

### 1. По расписанию (автоматически)
Бэкап выполняется раз в 3 дня в 03:00 (планировщик внутри приложения). Отдельной настройки не требует — работает, пока запущено приложение.

### 2. По команде в Telegram
Команда **`/dump`**, отправленная боту, запускает бэкап немедленно.

- Доступна только из разрешённых чатов: секретный канал админки (`TELEGRAM_CHANNEL_ID`) и канал дампов (`TELEGRAM_CHANNEL_ID_DUMP`). Из других чатов команда отклоняется.
- Бот отвечает «Делаю бэкап БД...», затем при успехе в канал дампов приходит файл, при ошибке — сообщение о причине.

### 3. По HTTP (из админ-панели)
**POST** `/api/admin/backup/run`

- Авторизация: требуется (администратор).
- Запускает бэкап немедленно, тем же путём, что и команда `/dump`.

**Поля ответа:**

| Поле | Тип | Описание |
|------|-----|----------|
| ok | булево | Успешно ли выполнен бэкап |
| sizeMb | число | Размер дампа в мегабайтах (при успехе) |
| error | строка | Описание ошибки (если не удалось) |

---

## Переменные окружения

| Переменная | Назначение |
|------------|-----------|
| DATABASE_URL | Строка подключения к БД (из неё берутся данные для pg_dump) |
| TELEGRAM_BOT_TOKEN | Токен бота, отправляющего дампы и алерты |
| TELEGRAM_CHANNEL_ID_DUMP | Канал, куда отправляются файлы дампов |
| TELEGRAM_CHANNEL_ID | Секретный канал админки (также разрешён для команды /dump) |

---

## Рекомендации по надёжности

- Файлы дампов в Telegram сохраняются даже при потере сервера — это внешняя копия.
- Локальная копия на сервере — быстрый доступ для восстановления.
- Периодически проверяй, что дамп реально разворачивается на тестовой базе. Бэкап, который не восстанавливается, бэкапом не является.
- Бэкап по расписанию работает только при запущенном приложении. Если приложение долго лежит, очередной автоматический бэкап не создастся (уже сделанные копии при этом не теряются).
