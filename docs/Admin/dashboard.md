# API админ-панели: Дашборд и статистика

**Базовый URL:** `/api/admin/dashboard`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

---

## 1. Общая сводка

**GET** `/api/admin/dashboard/overview`

Главная сводная статистика для главной страницы админки.

| Секция | Поля |
|--------|------|
| `revenue` | `total` (вся выручка, оплаченные не отменённые), `today`, `thisWeek`, `thisMonth`, `thisYear`, `averageOrderValue` (средний чек) |
| `orders` | `total`, `today`, `thisWeek`, `thisMonth`, `pending` (заказы, ожидающие оплаты) |
| `users` | `total`, `today`, `thisMonth` |
| `products` | `total`, `active`, `inactive` (деактивированные товары) |

---

## 2. График выручки по периодам

**GET** `/api/admin/dashboard/revenue`

Данные для построения графиков выручки.

| `period` | Период | Группировка |
|----------|--------|-------------|
| `day` | Последние 24 часа | по часу |
| `week` | Последние 7 дней | по дню |
| `month` (default) | Последние 30 дней | по дню |
| `year` | Последние 12 месяцев | по месяцу |

Ответ: `period`, `startDate`, `endDate`, `data` (массив точек с `date`, `revenue`, `orders`). Учитываются только оплаченные, не отменённые заказы. Недопустимое значение `period` → ошибка `400`.

---

## 3. Топ продаваемых товаров

**GET** `/api/admin/dashboard/top-products`

Список самых продаваемых товаров по количеству проданных единиц.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `limit` | number | 10 | Сколько товаров вернуть |

Ответ: массив товаров с полями `productId`, `name`, `slug`, `price`, `isActive`, `imageUrl`, `soldCount` (продано единиц из оплаченных не отменённых заказов), `revenue` (выручка с учётом скидки товара). Для удалённого товара — `name: "Товар удалён"`, `slug: null`.

---

## 4. Последние заказы

**GET** `/api/admin/dashboard/recent-orders`

Лента последних созданных заказов для быстрого просмотра.

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `limit` | number | 10 | Сколько заказов вернуть |

Ответ: массив заказов с полями `id`, `orderNumber`, `customerName`, `email`, `status`, `isPaid`, `total`, `currency`, `itemsCount`, `createdAt`. Полная информация о заказе — `GET /api/admin/orders/:orderNumber`.

---

## Примечания

- **Производительность:** все запросы дашборда параллельные (`Promise.all`) и используют агрегации Prisma. Для больших объёмов рекомендуется кэшировать `overview` на 1-2 минуты.
- **Что считается выручкой:** везде, где упоминается revenue/total/средний чек — учитываются только заказы с `isPaid: true` и `status != CANCELLED`.
