# API Промокодов (Admin)

Административный API для управления промокодами.

**Базовый URL:** `/api/admin/promo-codes`

**Авторизация:** все эндпоинты требуют авторизации администратора (`AdminGuard`). Запросы авторизуются через httpOnly cookie `adminToken`, которую браузер отправляет автоматически (заголовок `Authorization` также принимается для обратной совместимости).

---

## Типы данных

### PromoType (тип промокода)

| Значение | Описание |
|----------|----------|
| `PERCENTAGE` | Процентная скидка (10% = value: 10) |
| `FIXED` | Фиксированная скидка в рублях (500₽ = value: 500) |
| `FREE_DELIVERY` | Бесплатная доставка |

### PromoAppliesTo (к чему применяется)

| Значение | Описание |
|----------|----------|
| `ALL` | Ко всем товарам |
| `PRODUCT` | К конкретным товарам (по ID) |

### Объект PromoCode

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | Уникальный идентификатор |
| `code` | string | Код промокода (хранится в верхнем регистре) |
| `type` | string | Тип (`PERCENTAGE`, `FIXED`, `FREE_DELIVERY`) |
| `value` | number | Значение скидки (процент или сумма в RUB) |
| `appliesTo` | string | К чему применяется |
| `allowedUserIds` | string[] | Массив UUID пользователей (пустой = для всех) |
| `requiresAuth` | boolean | Требуется авторизация (только для зарегистрированных) |
| `maxUses` | number \| null | Общий лимит использований (`null` = безлимит) |
| `maxUsesPerUser` | number \| null | Лимит на одного пользователя (`null` = безлимит) |
| `usedCount` | number | Текущее количество использований |
| `maxItems` | number \| null | Максимум товаров, к которым применяется скидка (`null` = все) |
| `minOrderAmount` | number \| null | Минимальная сумма заказа для активации |
| `specificProductIds` | number[] | ID товаров, к которым применяется |
| `excludedProductIds` | number[] | ID товаров, исключённых из скидки |
| `excludeNewItems` | boolean | Не применять к товарам с cardStatus=NEW (по умолчанию true) |
| `validFrom` | string | Дата начала действия (ISO 8601) |
| `validUntil` | string | Дата окончания действия (ISO 8601) |
| `isActive` | boolean | Активен ли промокод |
| `description` | string | Автоматически сгенерированное описание |
| `createdAt` | string | Дата создания |
| `updatedAt` | string | Дата обновления |
| `createdBy` | string | Кем создан (по умолчанию `admin`) |

## Эндпоинты

### Создать промокод

**POST** `/api/admin/promo-codes`

Можно создавать промокоды с кодами, которые ранее использовались, если предыдущие промокоды с таким кодом деактивированы, истекли или исчерпали лимит использований.

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `code` | string | Да | Код (будет сохранён в верхнем регистре) |
| `type` | string | Да | `PERCENTAGE`, `FIXED` или `FREE_DELIVERY` |
| `value` | number | Да | Значение скидки (>= 0) |
| `appliesTo` | string | Нет | По умолчанию `ALL` |
| `allowedUserIds` | string[] | Нет | Массив UUID пользователей (пустой = для всех) |
| `requiresAuth` | boolean | Нет | По умолчанию `false` |
| `maxUses` | number | Нет | Общий лимит (>= 1, `null` = безлимит) |
| `maxUsesPerUser` | number | Нет | Лимит на пользователя (>= 1, `null` = безлимит) |
| `maxItems` | number | Нет | Макс. товаров для скидки (`null` = все) |
| `minOrderAmount` | number | Нет | Минимальная сумма заказа в рублях (>= 0) |
| `excludeNewItems` | boolean | Нет | Не применять к новинкам (по умолчанию true) |
| `specificProductIds` | number[] | Нет | ID товаров, к которым применяется |
| `excludedProductIds` | number[] | Нет | ID товаров-исключений |
| `validFrom` | string | Нет | Дата начала (ISO 8601) |
| `validUntil` | string | Нет | Дата окончания (ISO 8601) |
| `isActive` | boolean | Нет | По умолчанию `true` |
| `description` | string | Нет | Описание (для админки) |

Ответ: объект с полями `promoCode` (созданный промокод) и `description` (сгенерированное описание).

Примечания по сценариям:
- **FREE_DELIVERY:** поле `value` должно быть `0`. Промокод даёт только бесплатную доставку, не скидку на товары; автоматически `appliesTo = ALL`.
- **Конкретные товары:** `appliesTo = PRODUCT` + `specificProductIds`.
- **Только для зарегистрированных:** `requiresAuth = true` — доступен всем авторизованным, но не гостям.
- **Персональный/групповой:** заполненный `allowedUserIds` — доступен только указанным пользователям.
- **Новинки:** по умолчанию `excludeNewItems=true` — промокоды PERCENTAGE/FIXED не применяются к товарам с `cardStatus='NEW'`. FREE_DELIVERY всегда применяется к новинкам.
- **maxItems:** ограничивает количество товаров, к которым применяется скидка; скидка применяется к N самым дорогим товарам в корзине.

### Получить все промокоды

**GET** `/api/admin/promo-codes`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `isActive` | string | - | Фильтр по активности (`true` / `false`) |
| `type` | string | - | Фильтр по типу (`PERCENTAGE`, `FIXED`, `FREE_DELIVERY`) |
| `page` | string | 1 | Номер страницы |
| `limit` | string | 50 | Количество записей на странице |

Ответ: объект с полями `promoCodes` (массив) и `pagination` (`page`, `limit`, `total`, `totalPages`).

---

### Получить промокод по ID

**GET** `/api/admin/promo-codes/:id`

Возвращает промокод вместе с последними 10 использованиями (`usages`, каждое с вложенным `user`).

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID промокода |

Ошибки: `404` — промокод не найден.

---

| Поле | Тип | Описание |
|------|-----|----------|
| `code` | string | Новый код |
| `type` | string | `PERCENTAGE`, `FIXED` или `FREE_DELIVERY` |
| `value` | number | Значение скидки (>= 0) |
| `appliesTo` | string | К чему применяется |
| `allowedUserIds` | string[] | Массив UUID (пустой = для всех) |
| `requiresAuth` | boolean | Требуется авторизация |
| `maxUses` | number | Общий лимит (>= 1, `null` = безлимит) |
| `maxUsesPerUser` | number | Лимит на пользователя (>= 1, `null` = безлимит) |
| `maxItems` | number | Макс. товаров (`null` = все) |
| `minOrderAmount` | number | Минимальная сумма заказа (>= 0) |
| `excludeNewItems` | boolean | Не применять к новинкам |
| `specificProductIds` | number[] | ID товаров, к которым применяется |
| `excludedProductIds` | number[] | ID товаров-исключений |
| `validFrom` | string | Дата начала (ISO 8601) |
| `validUntil` | string | Дата окончания (ISO 8601) |
| `isActive` | boolean | Активность |
| `description` | string | Описание |

### Удалить промокод

**DELETE** `/api/admin/promo-codes/:id`

Полное удаление из базы данных.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID промокода |

Ответ: сообщение об удалении.

---

### Деактивировать промокод

**POST** `/api/admin/promo-codes/:id/deactivate`

Деактивация без удаления — устанавливает `isActive = false`.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID промокода |

Ответ: обновлённый объект промокода.

---

### Статистика использований

**GET** `/api/admin/promo-codes/:id/stats`

Статистика использований промокода по пользователям.

| Параметр пути | Тип | Описание |
|---------------|-----|----------|
| `id` | number | ID промокода |

Поля ответа: `promoCodeId`, `code`, `maxUses`, `maxUsesPerUser`, `totalUsedCount`, `userStats` (массив с вложенным `user` и `usageCount`).

Гостевые использования (`userId=null`) не включаются в `userStats`, но учитываются в `totalUsedCount`.

## UX Рекомендации для фронтенда

### Шаг 1: Тип промокода (`type`)

| Выбран тип | Поля для показа | Поля для скрытия |
|------------|-----------------|------------------|
| `PERCENTAGE` | `value` (1-100, label «Процент скидки %») | — |
| `FIXED` | `value` (label «Сумма скидки в ₽») | — |
| `FREE_DELIVERY` | — | `value`, `appliesTo`, `specificProductIds`, `excludedProductIds`, `maxItems`, `excludeNewItems` |

Для `FREE_DELIVERY` автоматически устанавливайте `value: 0`, `appliesTo: 'ALL'`; скрывайте все поля, кроме кода, пользователей, лимитов, дат, минимальной суммы.

### Шаг 2: К чему применяется (`appliesTo`)

Показывать только если `type` ≠ `FREE_DELIVERY`.

| Выбрано | Показывать | Скрывать | Откуда брать данные |
|---------|------------|----------|---------------------|
| `ALL` | `excludedProductIds` (опционально), `excludeNewItems` | `specificProductIds` | — |
| `PRODUCT` | `specificProductIds` (обязательно), `excludeNewItems` | — | `GET /api/products` |

`excludedProductIds` показывать при любом `appliesTo` — позволяет исключить товары из скидки.

### Шаг 3: Новинки (`excludeNewItems`)

Показывать только если `type` ≠ `FREE_DELIVERY`.

| Значение | Поведение |
|----------|-----------|
| `true` (по умолчанию) | Промокод НЕ применяется к товарам с cardStatus='NEW' |
| `false` | Промокод применяется ко всем товарам, включая новинки |

Чекбокс «Не применять к новинкам» (отмечен по умолчанию).

### Шаг 4: Для кого промокод (целевая аудитория)

| Вариант | Поля | Описание |
|---------|------|----------|
| Для всех (включая гостей) | `requiresAuth: false`, `allowedUserIds: []` | По умолчанию |
| Только для зарегистрированных (любых) | `requiresAuth: true`, `allowedUserIds: []` | Для всех авторизованных, но не для гостей |
| Только для конкретных пользователей | `requiresAuth: false`, `allowedUserIds: [...]` | Персональный промокод |

Важно: `requiresAuth: true` + `allowedUserIds: []` = для всех зарегистрированных; `requiresAuth: false` + `allowedUserIds: [...]` = только для конкретных пользователей; если оба активны, `allowedUserIds` имеет приоритет.

### Шаг 5: Лимиты использования

| Целевая аудитория | Показывать | Скрывать | Почему |
|-------------------|------------|----------|--------|
| Для всех (`requiresAuth: false`, `allowedUserIds: []`) | `maxUses`, `maxUsesPerUser` | — | Нужны оба лимита |
| Для зарегистрированных (`requiresAuth: true`, `allowedUserIds: []`) | `maxUses`, `maxUsesPerUser` | — | Нужны оба лимита |
| Для конкретных пользователей (`allowedUserIds: [...]`) | `maxUsesPerUser` | `maxUses` | Общий лимит = кол-во пользователей × maxUsesPerUser |

Валидация для публичных промокодов (`requiresAuth: false` и `allowedUserIds: []`):

| `maxUses` | `maxUsesPerUser` | Статус | Действие |
|-----------|------------------|--------|----------|
| `null` | `null` | Опасно | Предупреждение: «Промокод без лимитов — может быть использован неограниченно» |
| `1000` | `null` | Опасно | Предупреждение: «Один пользователь может использовать все 1000 раз» |
| `1000` | `1` | Безопасно | Максимум 1000 пользователей по 1 разу |
| `null` | `1` | Безопасно | Неограниченно пользователей, но каждый только 1 раз |

### Шаг 6: Дополнительные ограничения

| Поле | Когда показывать | Описание |
|------|------------------|----------|
| `minOrderAmount` | Всегда | Минимальная сумма заказа (в ₽) |
| `maxItems` | `type` ≠ `FREE_DELIVERY` | Макс. товаров для скидки (`null` = все). Применяется к N самым дорогим |
| `validFrom` | Всегда | Дата начала действия |
| `validUntil` | Всегда | Дата окончания действия |
| `isActive` | Всегда | Активен ли промокод |

## Коды ошибок

| Код | Описание |
|-----|----------|
| 400 | Bad Request — некорректные данные или дубликат активного кода |
| 401 | Unauthorized — отсутствует или невалидный токен |
| 403 | Forbidden — недостаточно прав (требуется AdminGuard) |
| 404 | Not Found — промокод не найден |
| 500 | Internal Server Error — внутренняя ошибка сервера |

## Примечания

1. **Код в верхнем регистре:** код автоматически преобразуется в верхний регистр при создании и обновлении.
2. **Автогенерация описания:** при создании/обновлении автоматически генерируется человекочитаемое описание.
3. **Повторное использование кодов:** допускается, если предыдущий промокод деактивирован, истёк или исчерпал лимит.
4. **Валидация:** все данные валидируются через `class-validator`; ошибки возвращаются с кодом 400.
5. **Ограничение доступа:** `requiresAuth: false` + пустой `allowedUserIds` = для всех (включая гостей); `requiresAuth: true` + пустой `allowedUserIds` = только зарегистрированные; заполненный `allowedUserIds` = только указанные пользователи. Гость при недоступном промокоде получает соответствующую ошибку.
6. **Лимиты:** `maxUses` — общий лимит, `maxUsesPerUser` — на пользователя; `null` = безлимит. Для промокодов с конкретными пользователями показывайте только `maxUsesPerUser`.
7. **Новинки:** по умолчанию `excludeNewItems=true`; FREE_DELIVERY всегда применяется к новинкам.
8. **FREE_DELIVERY:** даёт бесплатную доставку, `value` должно быть `0`, автоматически `appliesTo = ALL`.
9. **Пагинация:** по умолчанию до 50 записей на страницу.
10. **Авторизация:** все эндпоинты требуют `AdminGuard`. Запросы авторизуются через httpOnly cookie `adminToken` (заголовок `Authorization` также принимается для обратной совместимости).
