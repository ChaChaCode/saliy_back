# API админ-панели: Email-рассылки

**Базовый URL:** `/api/admin/campaigns`

**Требуется авторизация:** Admin Bearer Token

Массовая отправка email-писем пользователям. **Без Redis/Bull** — используется простой фоновый процесс с трекингом прогресса в БД и батчингом (20 писем за раз, задержка 1 сек между батчами чтобы не словить rate limit SMTP).

---

## Жизненный цикл кампании

```
DRAFT ──────────► SENDING ──────────► COMPLETED
                     │
                     └────────────────► FAILED
```

- **DRAFT** — создана, но не запущена. Можно редактировать/удалять.
- **SENDING** — идёт рассылка, прогресс обновляется в `sentCount`/`failedCount` каждый батч.
- **COMPLETED** — все письма обработаны (успешно или с ошибкой, но процесс завершён).
- **FAILED** — неожиданная ошибка, процесс прерван. Можно запустить заново (`POST /:id/send`).

---

## Типы аудитории

| `targetType` | Кому отправлять |
|--------------|-----------------|
| `ALL` (default) | Всем зарегистрированным |
| `WITH_ORDERS` | Только тем, у кого есть хотя бы один заказ |
| `WITHOUT_ORDERS` | Только тем, кто ещё не покупал |

---

## 1. Создать кампанию

**POST** `/api/admin/campaigns`

Создаёт черновик. Не начинает отправку.

### Request:
```json
{
  "subject": "Новая коллекция 2026!",
  "html": "<h1>Встречайте новинки</h1><p>Ссылка: <a href='https://saliy-shop.ru/new'>Смотреть</a></p>",
  "targetType": "WITH_ORDERS"
}
```

### Response:
```json
{
  "id": "campaign-uuid",
  "subject": "Новая коллекция 2026!",
  "html": "<h1>...</h1>",
  "targetType": "WITH_ORDERS",
  "status": "DRAFT",
  "totalCount": 0,
  "sentCount": 0,
  "failedCount": 0,
  "createdBy": "admin-uuid",
  "createdAt": "2026-04-11T12:00:00.000Z"
}
```

---

## 2. Запустить рассылку

**POST** `/api/admin/campaigns/:id/send`

Запускает отправку в фоне. Возвращается **немедленно** — реальная отправка идёт асинхронно.

### Response:
```json
{
  "success": true,
  "message": "Рассылка запущена в фоне",
  "totalRecipients": 315
}
```

### Ошибки:
- `400` — кампания уже в статусе `SENDING` / `COMPLETED`
- `400` — нет получателей (пустая выборка)

### Как отслеживать прогресс:
Опрашивайте `GET /api/admin/campaigns/:id` — поля `sentCount` и `failedCount` обновляются каждые ~20 писем.

---

## 3. Список кампаний

**GET** `/api/admin/campaigns?page=1&limit=20`

### Response:
```json
{
  "campaigns": [
    {
      "id": "uuid",
      "subject": "Новая коллекция 2026!",
      "status": "COMPLETED",
      "totalCount": 315,
      "sentCount": 310,
      "failedCount": 5,
      "startedAt": "2026-04-11T12:00:00.000Z",
      "completedAt": "2026-04-11T12:05:00.000Z",
      "createdAt": "2026-04-11T11:58:00.000Z"
    }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 12, "totalPages": 1 }
}
```

---

## 4. Получить кампанию

**GET** `/api/admin/campaigns/:id`

Полная инфа включая `html` и текущий прогресс. Используйте для построения прогресс-бара.

---

## 5. Удалить кампанию

**DELETE** `/api/admin/campaigns/:id`

### Ограничения:
- ❌ Нельзя удалить **активную** кампанию (`status: SENDING`) — сначала дождитесь завершения

---

## Технические детали

### Батчинг
Отправка идёт группами по **20 писем** с паузой **1 секунда** между батчами. Это снижает риск попадания в спам-листы SMTP-провайдера.

### Переменные для настройки:
- `BATCH_SIZE = 20` — в [src/admin/campaigns/campaigns.service.ts](src/admin/campaigns/campaigns.service.ts#L22)
- `BATCH_DELAY_MS = 1000` — там же

### Если сервер упадёт во время SENDING
Кампания останется в статусе `SENDING` с сохранённым прогрессом. Можно запустить повторно через `POST /:id/send` — рассылка начнётся заново (⚠️ адреса не дедуплицируются между запусками, пользователи могут получить дубль).

Для продакшна с большими объёмами (>10 000 писем) рекомендуется вынести в BullMQ с Redis.
