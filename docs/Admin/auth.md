| Способ       | Заголовок                       | Описание                             |
| ------------ | ------------------------------- | ------------------------------------ |

## Вход через Telegram

Авторизация через Telegram-кнопку. При запросе входа в защищённый Telegram-канал отправляется сообщение с кнопкой для входа.

### Поток авторизации

1. На фронте админки пользователь нажимает "Запросить вход"
2. Фронтенд отправляет `POST /api/admin/auth/request-login` и получает `loginId`
3. Фронтенд начинает поллить `GET /api/admin/auth/check-status/:loginId` каждые 2 секунды
4. В Telegram-канал приходит сообщение с кнопкой "Подтвердить вход"
5. Администратор нажимает кнопку в Telegram → вход подтверждается
6. Фронтенд получает `approved: true` + токен при следующем polling
7. Пользователь автоматически авторизуется

---

### Запрос входа

Генерирует JWT токен и отправляет кнопку входа в Telegram-канал.

**Эндпоинт:** `POST /api/admin/auth/request-login`

**Авторизация:** Не требуется

**Rate Limit:** 5 запросов в 10 минут (по IP)

#### Пример запроса

```http
POST /api/admin/auth/request-login
```

#### Пример ответа

```json
{
  "loginId": "a1b2c3d4e5f6789012345678abcdef01",
  "expiresIn": 300,
  "verificationCode": "X7K9"
}
```

#### Описание полей ответа

| Поле               | Тип    | Описание                                                               |
| ------------------ | ------ | ---------------------------------------------------------------------- |
| `loginId`          | string | Уникальный идентификатор запроса входа                                 |
| `expiresIn`        | number | Время жизни запроса в секундах (300 = 5 минут)                         |
| `verificationCode` | string | 4-символьный код для верификации (отображается на фронте и в Telegram) |

#### Сообщение в Telegram

После запроса в Telegram-канал придёт сообщение с кнопками:

```
🔐 Запрос входа в админку

🔑 Код: X7K9

📍 IP: 81.90.31.216
🌍 Локация: Москва, Россия
🏢 Провайдер: Rostelecom
🌐 Браузер: Chrome 142
🕐 Время (МСК): 15.02.2026, 01:50:00

⏱ Ссылка действительна 5 минут

[Подтвердить]  ← зелёная кнопка (style: success)
[Блок IP]      ← красная кнопка (style: danger)
```

**Код верификации** (`X7K9`) - отображается и на фронте, и в Telegram. Сверьте коды перед подтверждением.

#### Навигация кнопок

Все кнопки имеют цветовую индикацию:

- **Зелёные (success)** — позитивные действия (подтвердить, разблокировать)
- **Красные (danger)** — опасные действия (заблокировать, отозвать)

##### После подтверждения входа:

```
[Выполнено]              ← зелёная (неактивная)
[Блок IP] [Отозвать]  ← красные
```

- **Блок IP** — заблокировать IP даже после успешного входа
- **Отозвать** — отозвать выданный токен (пользователь будет разлогинен)

##### После блокировки IP:

```
[Заблокирован]     ← красная (неактивная)
[🔓 Разблокировать]   ← зелёная
```

##### После разблокировки IP:

```
[Разблокирован]  ← зелёная (неактивная)
[Блок IP]        ← красная (можно снова заблокировать)
```

##### После отзыва токена:

```
[Отозван]    ← красная (неактивная)
[Блок IP]    ← красная
```

#### Отзыв токена

Если вы случайно подтвердили вход злоумышленнику или хотите прервать сессию:

1. Нажмите кнопку **"Отозвать"** в Telegram
2. Токен добавляется в blacklist
3. Все последующие запросы с этим токеном будут отклонены (401 Unauthorized)
4. Пользователь будет разлогинен при следующем запросе

**Важно:** Отозванные токены хранятся в памяти 24 часа (время жизни токена), после чего автоматически удаляются

#### Ошибки

| Код | Сообщение              | Описание                               |
| --- | ---------------------- | -------------------------------------- |
| 429 | Слишком много запросов | Превышен лимит (5 запросов в 10 минут) |

При ошибке 429 в ответе также будет поле `retryAfter` с количеством секунд до сброса лимита.

### Проверка статуса входа

Проверяет, была ли нажата кнопка входа в Telegram.

**Эндпоинт:** `GET /api/admin/auth/check-status/:loginId`

**Авторизация:** Не требуется

#### Пример запроса

```http
GET /api/admin/auth/check-status/a1b2c3d4e5f6789012345678abcdef01
```

#### Пример ответа (ожидание)

```json
{
  "approved": false
}
```

#### Пример ответа (вход подтверждён)

```json
{
  "approved": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

#### Описание полей ответа

| Поле       | Тип     | Описание                                |
| ---------- | ------- | --------------------------------------- |
| `approved` | boolean | Был ли вход подтверждён через Telegram  |
| `token`    | string  | JWT токен (только при `approved: true`) |

#### Ошибки

| Код | Сообщение                  | Описание                                  |
| --- | -------------------------- | ----------------------------------------- |
| 401 | Запрос не найден или истёк | Неверный loginId или прошло более 5 минут |
| 401 | Время ожидания истекло     | Прошло более 5 минут с момента запроса    |

## Использование токена

После получения токена используйте его в заголовке `Authorization`:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Пример использования на фронтенде

```javascript
// 1. Запросить вход
const requestLogin = async () => {
  const response = await fetch('/api/admin/auth/request-login', {
    method: 'POST',
  });
  return response.json();
};

// 2. Polling для проверки статуса
const pollLoginStatus = async loginId => {
  return new Promise((resolve, reject) => {
    const interval = setInterval(async () => {
      try {
        const response = await fetch(`/api/admin/auth/check-status/${loginId}`);
        const data = await response.json();

        if (data.approved) {
          clearInterval(interval);
          resolve(data.token);
        }
      } catch (error) {
        clearInterval(interval);
        reject(error);
      }
    }, 2000); // Проверяем каждые 2 секунды

    // Таймаут через 5 минут
    setTimeout(
      () => {
        clearInterval(interval);
        reject(new Error('Timeout'));
      },
      5 * 60 * 1000,
    );
  });
};

// 3. Полный поток авторизации
const login = async () => {
  const { loginId } = await requestLogin();
  console.log('Ожидание подтверждения в Telegram...');

  const token = await pollLoginStatus(loginId);
  localStorage.setItem('adminToken', token);

  console.log('Вход выполнен!');
};

// 4. Использование токена для запросов
const fetchOrders = async () => {
  const token = localStorage.getItem('adminToken');
  const response = await fetch('/api/admin/orders', {
    headers: {
      Authorization: `Bearer ${token}`,
    },
  });
  return response.json();
};
```

### Время жизни токена

| Параметр                  | Значение |
| ------------------------- | -------- |
| Время жизни токена        | 24 часа  |
| Время жизни запроса входа | 5 минут  |

### Обновление токена (refresh)

Выдаёт новый токен на 24 часа, если текущий ещё валиден.

**Эндпоинт:** `POST /api/admin/auth/refresh`

**Авторизация:** Требуется (Bearer Token)

#### Пример запроса

```http
POST /api/admin/auth/refresh
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

#### Пример ответа

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 86400
}
```

#### Описание полей ответа

| Поле        | Тип    | Описание                          |
| ----------- | ------ | --------------------------------- |
| `token`     | string | Новый JWT токен                   |
| `expiresIn` | number | Время жизни токена в секундах (24 часа) |

#### Пример использования на фронтенде

```javascript
// Обновление токена (рекомендуется вызывать каждые 12 часов или при старте)
const refreshToken = async () => {
  const currentToken = localStorage.getItem('adminToken');

  const response = await fetch('/api/admin/auth/refresh', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${currentToken}`,
    },
  });

  if (response.ok) {
    const { token } = await response.json();
    localStorage.setItem('adminToken', token);
    return token;
  }

  // Токен истёк или отозван — нужно перелогиниться через Telegram
  localStorage.removeItem('adminToken');
  window.location.href = '/login';
};

// Вызов при старте приложения
refreshToken();

// Или по интервалу (каждые 12 часов)
setInterval(refreshToken, 12 * 60 * 60 * 1000);
```

#### Ошибки

| Код | Сообщение                           | Описание                                  |
| --- | ----------------------------------- | ----------------------------------------- |
| 401 | Токен отозван                       | Токен был отозван через Telegram          |
| 401 | Невалидный токен                    | Токен истёк или невалиден                 |
| 401 | Администратор не найден или деактивирован | Администратор удалён или деактивирован |

### Ошибки токена

| Код | Сообщение                   | Описание                                  |
| --- | --------------------------- | ----------------------------------------- |
| 401 | Token has been revoked      | Токен был отозван через Telegram          |
| 401 | Invalid or expired token    | Токен истёк или невалиден                 |
| 401 | Invalid token type          | Токен не является админским               |
| 401 | Admin not found or inactive | Администратор не найден или деактивирован |

## Безопасность

- Кнопка входа отправляется только в приватный Telegram-канал
- Запрос действителен только 5 минут
- В сообщении указывается IP, геолокация и браузер для идентификации запроса
- При подозрительном запросе администратор может просто не нажимать кнопку
- JWT токен содержит привязку к конкретному запросу (loginId)
- Rate limit на запрос входа: 5 попыток в 10 минут с одного IP
- Защита от брутфорса: 10 неудачных попыток авторизации → блокировка IP на 15 минут
- **Отзыв токена** — возможность мгновенно отозвать выданный токен через Telegram
- **Блокировка после входа** — можно заблокировать IP даже после подтверждения входа
- **Циклическая навигация** — после любого действия всегда доступно обратное действие

## Защита от брутфорса

Все защищённые эндпоинты админ-панели (`/api/admin/*`) имеют защиту от перебора:

| Параметр                   | Значение                 |
| -------------------------- | ------------------------ |
| Максимум неудачных попыток | 10                       |
| Время блокировки           | 15 минут                 |
| Сброс счётчика             | При успешной авторизации |

### Как работает

1. При каждой неудачной попытке авторизации (401) счётчик увеличивается
2. После 10 неудачных попыток IP блокируется на 15 минут
3. При успешной авторизации счётчик сбрасывается
4. Блокировка автоматически снимается через 15 минут

### Уведомление в Telegram

При достижении лимита неудачных попыток в Telegram-канал приходит уведомление:

```
⚠️ Подозрительная активность

🚨 Попытка брутфорса админки!

📍 IP: 77.222.101.28
🌍 Локация: Москва, Россия
🏢 Провайдер: Rostelecom
🌐 Браузер: Chrome 142
🔢 Попыток: 10
🕐 Время (МСК): 15.02.2026, 21:30:00

⏱ IP временно заблокирован на 15 минут

[🔓 Разблокировать] [🚫 Заблокировать навсегда]
```

**Кнопки:**
- `🔓 Разблокировать` (style: success) — снять временную блокировку если это не атака
- `🚫 Заблокировать навсегда` (style: danger) — заблокировать IP навсегда в БД

При нажатии "Заблокировать навсегда":
- IP блокируется **навсегда** (сохраняется в БД)
- Кнопки меняются на "Заблокирован навсегда" + "🔓 Разблокировать"
- Можно разблокировать если заблокировали по ошибке

### Ошибка при блокировке

```json
{
  "message": "Слишком много неудачных попыток. Попробуйте позже.",
  "error": "Too Many Failed Attempts",
  "statusCode": 429,
  "retryAfter": 847
}
```

Поле `retryAfter` содержит количество секунд до разблокировки

## Управление заблокированными IP

### Типы блокировок

| Тип        | Хранение | Длительность | Как снять                            |
| ---------- | -------- | ------------ | ------------------------------------ |
| Временная  | В памяти | 15 минут     | Автоматически или перезапуск сервера |
| Постоянная | В БД     | Навсегда     | Через API или кнопку в Telegram      |

- **Временная блокировка** — срабатывает после 10 неудачных попыток авторизации
- **Постоянная блокировка** — при нажатии "Заблокировать IP" в Telegram

---

### Получить список заблокированных IP

**Эндпоинт:** `GET /api/admin/auth/blocked-ips`

**Авторизация:** Требуется (AdminGuard)

#### Пример ответа

```json
[
  {
    "id": 1,
    "ip": "77.222.101.28",
    "reason": "Заблокировано через Telegram",
    "userAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)...",
    "location": "Москва, Россия (Rostelecom)",
    "blockedAt": "2026-02-15T06:45:00.000Z",
    "isActive": true
  }
]
```

---

### Разблокировать IP

**Эндпоинт:** `POST /api/admin/auth/unblock/:ip`

**Авторизация:** Требуется (AdminGuard)

#### Пример запроса

```http
POST /api/admin/auth/unblock/77.222.101.28
```

#### Пример ответа

```json
{
  "success": true,
  "message": "IP 77.222.101.28 разблокирован"
}
```

---

### Удалить IP из базы

**Эндпоинт:** `DELETE /api/admin/auth/blocked-ips/:ip`

**Авторизация:** Требуется (AdminGuard)

#### Пример запроса

```http
DELETE /api/admin/auth/blocked-ips/77.222.101.28
```

#### Пример ответа

```json
{
  "success": true,
  "message": "IP 77.222.101.28 удалён из базы"
}
```
