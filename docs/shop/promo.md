# API промокодов (публичное)

## Валидация промокода

**POST** `/api/promo/validate`

Проверяет валидность промокода и рассчитывает скидку.

**Авторизация:** Опциональна (работает для гостей и авторизованных)

### Request:
```json
{
  "code": "SALE10",
  "orderAmount": 9500,
  "cartItems": [
    {
      "productId": 20,
      "quantity": 1,
      "price": 9500
    }
  ]
}
```

### Параметры:
- **code** (обязательно) - код промокода
- **orderAmount** (опционально) - сумма заказа для проверки минимального порога
- **cartItems** (опционально) - товары в корзине для расчета скидки

### Response (валидный промокод):

#### Процентная скидка (PERCENTAGE):
```json
{
  "isValid": true,
  "discount": 950,
  "message": "Скидка 10%",
  "promoCode": {
    "id": 1,
    "code": "SALE10",
    "type": "PERCENTAGE",
    "value": 10,
    "description": "Скидка 10% на все товары"
  }
}
```

#### Фиксированная скидка (FIXED):
```json
{
  "isValid": true,
  "discount": 500,
  "message": "Скидка 500 ₽",
  "promoCode": {
    "id": 2,
    "code": "SAVE500",
    "type": "FIXED",
    "value": 500,
    "description": "Фиксированная скидка 500₽"
  }
}
```

#### Бесплатная доставка (FREE_DELIVERY):
```json
{
  "isValid": true,
  "discount": 0,
  "isFreeDelivery": true,
  "message": "Бесплатная доставка",
  "promoCode": {
    "id": 3,
    "code": "FREESHIP",
    "type": "FREE_DELIVERY",
    "value": 0,
    "description": "Бесплатная доставка",
    "isFreeDelivery": true
  }
}
```

### Response (невалидный промокод):
```json
{
  "isValid": false,
  "discount": 0,
  "message": "Промокод не найден"
}
```

### Возможные причины невалидности:

| Сообщение | Причина |
|-----------|---------|
| `Промокод не найден` | Такого промокода не существует |
| `Промокод не активен` | Промокод деактивирован администратором |
| `Промокод начнёт действовать DD.MM.YYYY` | Промокод еще не начал действовать |
| `Срок действия промокода истек` | Промокод истек по дате validUntil |
| `Промокод больше не действителен` | Достигнут максимальный лимит использований (maxUses) |
| `Промокод недоступен для вас` | Промокод доступен только для определенных пользователей |
| `Вы уже использовали этот промокод максимальное количество раз` | Достигнут лимит на пользователя (maxUsesPerUser) |
| `Минимальная сумма заказа: XXXX ₽` | Сумма заказа меньше minOrderAmount |
| `Промокоды не применяются к заказам с новинками (Название товара)` | В корзине есть товары с меткой NEW, промокод не FREE_DELIVERY |
| `Промокод не применим к товарам в корзине` | Нет подходящих товаров (все исключены или не входят в specificProductIds) |

## Применение промокода в заказе

Промокод применяется автоматически при создании заказа, если передать поле `promoCode`:

**POST** `/api/orders`

```json
{
  "items": [...],
  "firstName": "Иван",
  "lastName": "Петров",
  "email": "test@example.com",
  "phone": "+375291234567",
  "deliveryType": "STANDARD",
  "paymentMethod": "CARD_ONLINE",
  "promoCode": "SALE10"
}
```

### Поведение:
- ✅ Промокод валидируется на сервере
- ✅ Если невалиден - возвращается ошибка 400 с причиной
- ✅ Если валиден - скидка применяется автоматически
- ✅ Использование записывается в историю

## Логика работы промокодов

### Типы скидок:
1. **PERCENTAGE** - процентная скидка (10% = value: 10)
2. **FIXED** - фиксированная скидка в рублях (500₽ = value: 500)
3. **FREE_DELIVERY** - бесплатная доставка (скидка на доставку)

### Ограничения применения:

#### По товарам:
- **appliesTo: ALL** - применяется ко всем товарам
- **appliesTo: PRODUCT** - только к определенным товарам (specificProductIds)
- **excludedProductIds** - исключенные товары (не применяется)

#### По новинкам:
❌ **PERCENTAGE и FIXED не применяются к товарам с меткой NEW**
✅ **FREE_DELIVERY применяется к любым товарам**

#### Лимит на количество товаров:
- **maxItems** - скидка применяется только к N самым дорогим товарам
- Пример: maxItems=2 → скидка на 2 самых дорогих товара в корзине

### Ограничения по использованию:
- **maxUses** - общее количество использований промокода
- **maxUsesPerUser** - сколько раз один пользователь может использовать
- **minOrderAmount** - минимальная сумма заказа в рублях
- **validFrom / validUntil** - период действия промокода
- **allowedUserIds** - список UUID пользователей (пустой = для всех)

### Приоритет применения:
1. Проверяется активность и срок действия
2. Проверяются лимиты использований
3. Проверяется доступность для пользователя
4. Проверяется минимальная сумма заказа
5. Проверяются товары (наличие новинок, исключения)
6. Рассчитывается применимая сумма
7. Применяется скидка

## Примеры использования

### Пример 1: Проверка промокода перед оформлением заказа

```bash
# 1. Проверить промокод
curl -X POST https://saliy-shop.ru/api/promo/validate \
  -H "Content-Type: application/json" \
  -d '{
    "code": "SALE10",
    "orderAmount": 9500,
    "cartItems": [
      {"productId": 20, "quantity": 1, "price": 9500}
    ]
  }'

# Response:
# {
#   "isValid": true,
#   "discount": 950,
#   "message": "Скидка 10%"
# }
```

### Пример 2: Применение промокода в заказе

```bash
# 2. Оформить заказ с промокодом
curl -X POST https://saliy-shop.ru/api/orders \
  -H "Content-Type: application/json" \
  -d '{
    "items": [{"productId": 20, "size": "M", "quantity": 1}],
    "firstName": "Иван",
    "lastName": "Петров",
    "email": "test@example.com",
    "phone": "+375291234567",
    "deliveryType": "STANDARD",
    "paymentMethod": "CARD_ONLINE",
    "promoCode": "SALE10"
  }'

# Response: заказ с примененной скидкой
# {
#   "id": "uuid",
#   "orderNumber": "260329-0001",
#   "subtotal": 9500,
#   "discountAmount": 950,
#   "total": 8550,
#   ...
# }
```

### Пример 3: Обработка ошибок

```bash
curl -X POST https://saliy-shop.ru/api/promo/validate \
  -H "Content-Type: application/json" \
  -d '{
    "code": "EXPIRED",
    "orderAmount": 1000
  }'

# Response:
# {
#   "isValid": false,
#   "discount": 0,
#   "message": "Срок действия промокода истек"
# }
```

## Интеграция на фронтенде

### Рекомендуемый flow:

1. **Пользователь вводит промокод**
2. **Вызвать POST /api/promo/validate** (опционально, для предварительной проверки)
3. **Показать результат:**
   - ✅ Валиден: "Промокод применен! Скидка 950 ₽"
   - ❌ Невалиден: показать сообщение с причиной
4. **Добавить promoCode в запрос создания заказа**
5. **Обработать возможную ошибку** (если промокод стал невалидным между проверкой и созданием)

### Без предварительной проверки (упрощенный flow):

1. Пользователь вводит промокод
2. Сразу добавляем в запрос создания заказа
3. Если ошибка - показываем причину

---

## Коды ошибок

| Код | Описание |
|-----|----------|
| 200 | Промокод валидирован успешно |
| 400 | Промокод невалиден (см. message) |
| 401 | Требуется авторизация (для промокодов с allowedUserIds) |
| 500 | Ошибка сервера |
