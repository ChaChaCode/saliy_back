# API доставки (CDEK)

Интеграция с CDEK для расчёта стоимости доставки, выбора пунктов выдачи и создания заказов.

Глобальный префикс приложения — `/api`.

---

## Авторизация

Клиентские эндпоинты доставки (countries, regions, cities, pickup-points, prices) публичны и авторизации не требуют. Сохранение адреса в профиле выполняется через `/api/auth/delivery-location` — там токен берётся из httpOnly cookie `accessToken` автоматически (заголовок Authorization тоже принимается для обратной совместимости).

---

## Поддерживаемые страны

CDEK доступен для:
- Россия (RU)
- Беларусь (BY)

Самовывоз CDEK доступен только для России и Беларуси. Почтовая (стандартная) доставка доступна для любой страны, включая Россию и Беларусь — там она идёт как дополнительный вариант к CDEK.

---

## Эндпоинты

### 1. Получить список стран

**GET** `/api/delivery/countries?lang=ru`

Возвращает список всех стран с типами доставки.

Query-параметры:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| lang | string | нет | Язык (ru/en/pl, по умолчанию ru) |

Поля каждого элемента ответа:

| Поле | Тип | Описание |
|------|-----|----------|
| code | string | Код страны (RU, BY, PL, …) |
| name | string | Название страны |
| deliveryTypes | string[] | Доступные типы доставки: `CDEK_PICKUP` и/или `STANDARD` |

---

### 2. Получить информацию о стране

**GET** `/api/delivery/countries/:code?lang=ru`

Возвращает один объект страны (поля `code`, `name`, `deliveryTypes`). Если страна не найдена — возвращается объект с полем `error: "Country not found"`.

---

### 3. Получить регионы страны

**GET** `/api/delivery/regions?countryCode=RU`

Query-параметры:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| countryCode | string | да | Код страны |

Если `countryCode` не передан — возвращается объект с полем `error`. Поля каждого региона:

| Поле | Тип | Описание |
|------|-----|----------|
| code | number | Код региона CDEK |
| name | string | Название региона |
| countryCode | string | Код страны |

---

### 4. Получить города

**GET** `/api/delivery/cities?countryCode=RU&regionCode=77&search=Москва`

Query-параметры:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| countryCode | string | да | Код страны |
| regionCode | number | нет | Код региона |
| search | string | нет | Поиск по названию города |

Для стран CDEK (RU, BY) обязателен либо `regionCode`, либо `search` — иначе сервер вернёт объект с полем `error` и подсказкой получить сначала список регионов. Поля каждого города:

| Поле | Тип | Описание |
|------|-----|----------|
| code | number | Код города CDEK |
| name | string | Название города |
| regionCode | number | Код региона |
| region | string | Название региона |
| countryCode | string | Код страны |

---

### 5. Получить пункты выдачи CDEK

**GET** `/api/delivery/pickup-points?cityCode=44`

Query-параметры:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| cityCode | number | да | Код города CDEK |

Поля каждого пункта выдачи:

| Поле | Тип | Описание |
|------|-----|----------|
| code | string | Код ПВЗ |
| name | string | Название ПВЗ |
| address | string | Адрес |
| city | string | Город |
| coordinates | number[] | Координаты [долгота, широта] |
| workTime | string | Часы работы |
| phones | string[] | Телефоны |
| isDressingRoom | boolean | Наличие примерочной |
| haveCashless | boolean | Приём безналичной оплаты |

---

### 6. Рассчитать стоимость доставки

**GET** `/api/delivery/prices?cityCode=44&weight=500&currency=RUB`

Рассчитывает стоимость доставки для самовывоза.

Query-параметры:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| cityCode | number | да | Код города CDEK |
| weight | number | нет | Вес товара в граммах (по умолчанию 500) |
| currency | string | нет | Валюта (по умолчанию RUB) |

Ответ содержит объект `pickup` со следующими полями:

| Поле | Тип | Описание |
|------|-----|----------|
| tariffCode | number | Код тарифа |
| tariffName | string | Название тарифа |
| tariffDescription | string | Описание тарифа |
| deliverySum | number | Реальная стоимость доставки от CDEK API (зависит от города и веса) |
| periodMin / periodMax | number | Срок доставки в рабочих днях |
| calendarMin / calendarMax | number | Срок доставки в календарных днях |
| currency | string | Валюта |

> Важно: этот же расчёт используется на бэке при создании заказа — клиент платит ровно столько, сколько здесь увидел. Если CDEK API недоступен в момент оформления, бэк использует fallback из админских настроек (`delivery_price_cdek`, по умолчанию 500₽).

---

### 7. Информация о заказе CDEK

**GET** `/api/delivery/orders/:uuid`

Возвращает информацию о заказе CDEK по его UUID. При ошибке возвращается объект с полями `error` и `message`.

---

### 8. URL отслеживания заказа

**GET** `/api/delivery/tracking/:cdekNumber`

Возвращает объект с полями `cdekNumber` и `trackingUrl`.

---

### 9. Webhook CDEK

**POST** `/api/delivery/webhook`

Принимает обновления статусов от CDEK. Тело — произвольный payload CDEK. В ответе возвращается `success: true` и результат обработки, либо `success: false` с полем `error`.

## Сценарий выбора доставки

### Шаг 1: выбрать страну

Запросить **GET** `/api/delivery/countries?lang=ru`. Пользователь выбирает страну; в поле `deliveryTypes` указаны доступные типы доставки.

Логика фронтенда:
- если `deliveryTypes` содержит `CDEK_PICKUP` → Вариант А (выбор через селекты, самовывоз из ПВЗ);
- иначе → Вариант Б (ввод адреса вручную, почтовая доставка).

### Вариант А: страны с CDEK (RU, BY)

Шаг 2А — выбрать регион и город:
1. Получить регионы: **GET** `/api/delivery/regions?countryCode=RU`.
2. Получить города региона (обязательно указать regionCode): **GET** `/api/delivery/cities?countryCode=RU&regionCode=81`.

Альтернатива — поиск по названию (без выбора региона): **GET** `/api/delivery/cities?countryCode=RU&search=Москва`.

Важно: без `regionCode` или `search` для стран CDEK API вернёт ошибку, так как список из 1500+ городов неудобен для выбора.

Шаг 3А — сохранить в профиле через **PUT** `/api/auth/delivery-location` с `deliveryType: 'CDEK'` и `cdekCityCode`. После этого в профиле автоматически заполнятся: `deliveryType` = CDEK, `cdekCityCode`, `cdekCountryCode`, `cdekRegionCode`, `cityName`, `countryName`, `regionName`.

### Вариант Б: другие страны (без CDEK)

Шаг 2Б — фронтенд показывает поля «полный адрес одной строкой» (регион, город, улица, дом, квартира) и «почтовый индекс».

Шаг 3Б — сохранить в профиле через **PUT** `/api/auth/delivery-location` с `deliveryType: 'POST'`, `deliveryCountryCode`, `fullAddress`, `postalCode`. После этого в профиле заполнятся: `deliveryType` = POST, `deliveryCountryCode`, `countryName`, `fullAddress`, `postalCode`.

### Шаг 4: рассчитать стоимость доставки

Запросить **GET** `/api/delivery/prices?cityCode=44&weight=650`.

### Шаг 5: выбрать способ доставки

- Для России/Беларуси (самовывоз из ПВЗ CDEK): получить список ПВЗ через **GET** `/api/delivery/pickup-points?cityCode=44`, пользователь выбирает ПВЗ и сохраняет его код.
- Для других стран (почтовая доставка): пользователь вводит адрес в профиле через **PUT** `/api/auth/delivery-location`.

| Поле | Тип | Описание |
|------|-----|----------|
| cdekCityCode | number | Код города CDEK (заполняется автоматически) |
| cdekCountryCode | string | Код страны CDEK |
| cdekRegionCode | number | Код региона CDEK |
| cityName | string | Название города |
| countryName | string | Название страны |
| regionName | string | Название региона |
| deliveryCountryCode | string | Код страны для POST |
| fullAddress | string | Полный адрес одной строкой (для POST) |
| postalCode | string | Почтовый индекс (для POST) |

## Типы доставки

| Тип | Код | Цена | Страны |
|-----|-----|------|--------|
| Самовывоз из ПВЗ CDEK | `CDEK_PICKUP` | CDEK API по городу + весу. Fallback — `delivery_price_cdek` (по умолчанию 500₽) | Россия, Беларусь |
| Почтовая доставка | `STANDARD` | Фикс из админских настроек (`delivery_price_standard`, по умолчанию 800₽) | Любая страна |

Почтовая доставка доступна для любой страны. Для России и Беларуси клиенту дополнительно предлагается самовывоз из ПВЗ CDEK — то есть для этих стран на выбор два варианта (CDEK и почта), для остальных стран только почта.

При создании заказа с почтовой доставкой (`deliveryType=STANDARD`) backend требует три поля: страну (`countryName`), полный адрес одной строкой (`street` — город, улица, дом, квартира) и почтовый индекс (`postalCode`). Если чего-то не хватает — заказ не создаётся, возвращается ошибка с перечислением недостающих полей. Для самовывоза CDEK эти поля не требуются (там код ПВЗ, адрес пункта подтягивается из CDEK автоматически).

## Пример UI flow

1. Регистрация/вход.
2. Заполнение профиля.
3. Выбор страны доставки: **GET** `/api/delivery/countries`.
4. Если RU/BY — выбор города: **GET** `/api/delivery/cities?search=...`.
5. Сохранение города в профиле: **PUT** `/api/auth/delivery-location`.
6. Добавление товаров в корзину.
7. Оформление заказа: расчёт веса товаров и получение цен доставки **GET** `/api/delivery/prices?cityCode=44&weight=650`.
8. Если Россия/Беларусь (CDEK_PICKUP): выбор ПВЗ **GET** `/api/delivery/pickup-points?cityCode=44`. Если другие страны (STANDARD): заполнение адреса **PUT** `/api/auth/delivery-location`.
9. Создание заказа.
10. Оплата (Яндекс Пей — автоуспех).

## Расчёт стоимости (как это работает на бэке)

При вызове **POST** `/api/orders/calculate` или **POST** `/api/orders` бэк:

1. Вес фиксированный — 500 г на любой заказ (по бизнес-решению, см. [src/orders/orders.service.ts](../../src/orders/orders.service.ts) — поле `weight` у товара не используется в расчёте доставки).
2. Если `deliveryType === 'CDEK_PICKUP'` и передан `cdekCityCode`:
   - Шаг A. Точечный запрос к CDEK `/v2/calculator/tariff` с `tariff_code = 136` («Посылка склад-склад»). Если CDEK вернул цену — берём её.
   - Шаг B (fallback). Если CDEK ответил 400 (например, тариф недоступен для маршрута) — вызывается `/v2/calculator/tarifflist`, и среди всех тарифов выбирается самый дешёвый «склад-склад», либо «X-склад» (последняя нога в ПВЗ).
   - Шаг C (fallback). Если ни один тариф не подошёл — берётся фикс из админских настроек `delivery_price_cdek` (по умолчанию 500₽).
3. Если `deliveryType === 'STANDARD'` → фикс из `delivery_price_standard` (по умолчанию 800₽).

В логах backend'а можно увидеть:
- успешный реальный расчёт через тариф 136 (например, `CDEK price: city=259, weight=500г → 338 ₽ [тариф 136: Посылка склад-склад]`);
- предупреждение о fallback на tarifflist (шаг B);
- предупреждение `Fallback 500 ₽` (шаг C — CDEK не дал ни одного подходящего тарифа).

### Поле response `deliveryPrice`

В ответе `/api/orders/calculate` и `/api/orders` поле `deliveryPrice` (внутри объекта суммы заказа) содержит итоговую цену, которую заплатит клиент. Это то же число, что вернёт **GET** `/api/delivery/prices?cityCode=...&weight=...` для тех же параметров.

## Важные замечания

### Вес посылки
- Фиксирован на стороне backend'а: 500 г для всех заказов. Поле `weight` у товара в БД сейчас на расчёт доставки не влияет.
- Если бизнес-логика поменяется — настройка в [src/orders/orders.service.ts](../../src/orders/orders.service.ts) (`const weight = 500;`).

### Склад отправителя

Настраивается через переменные окружения:

| Переменная | Описание |
|------------|----------|
| CDEK_WAREHOUSE_CITY_CODE | Код города отправителя в системе CDEK (НЕ телефонный код): Москва = 44, Санкт-Петербург = 137, Брест = 9220 |
| CDEK_PICKUP_TARIFF_CODE | Код тарифа склад-склад (по умолчанию 136 «Посылка склад-склад») |
| CDEK_SENDER_PICKUP_POINT | Конкретный ПВЗ-отправитель (опционально). Если CDEK возвращает err_pvz_with_tariff_mistake — укажите код своего ПВЗ или адрес склада |

> Внимание: `CDEK_WAREHOUSE_CITY_CODE` — это код города в системе CDEK, а не телефонный код. Москва = 44, не 495. Узнать код своего города можно через **GET** `/api/delivery/cities?countryCode=RU&search=Москва`, поле `code` в ответе.

### Тестовый режим
- `CDEK_TEST_MODE=true` → тестовый API `https://api.edu.cdek.ru/v2`, тестовые креды `CDEK_CLIENT_ID_TEST` / `CDEK_CLIENT_SECRET_TEST`.
- В тестовой среде у CDEK ограниченное покрытие тарифов — для многих маршрутов нет тарифа «склад-склад». На прод-API (`https://api.cdek.ru/v2`) обычно всё доступно.
