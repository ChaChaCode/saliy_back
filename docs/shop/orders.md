# API заказов

Базовый URL: `https://saliyclothes.com`. Все маршруты ниже указаны с глобальным префиксом `/api`.

## Авторизация

Для эндпоинтов, требующих авторизации, access token хранится в httpOnly cookie `accessToken` и отправляется браузером автоматически (заголовок `Authorization: Bearer` также принимается для обратной совместимости). Создание заказа и расчёт доступны и гостям, и авторизованным пользователям.

---

## 1. Получить доступные опции доставки и оплаты

**GET** `/api/orders/delivery-options`

Возвращает доступные типы доставки и методы оплаты для выбранной страны.

Query параметры:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| country | string | Да | Код страны (RU, BY, PL и т.д.) |

Правила:
- Россия и Беларусь (RU, BY): только `CDEK_PICKUP` (самовывоз из ПВЗ).
- Другие страны: только `STANDARD` (почтовая доставка).
- Все страны: метод оплаты `CARD_ONLINE` — онлайн-оплата картой через Альфа-Банк (см. [payment.md](./payment.md)).

Ответ: объект с полями `deliveryTypes` (массив доступных типов доставки), `paymentMethods` (массив доступных методов оплаты) и `country` (переданный код страны).

Примеры:
- Для `country=RU`: `deliveryTypes = ["CDEK_PICKUP"]`, `paymentMethods = ["CARD_ONLINE"]`.
- Для `country=PL`: `deliveryTypes = ["STANDARD"]`, `paymentMethods = ["CARD_ONLINE"]`.

## 2. Рассчитать стоимость заказа

**POST** `/api/orders/calculate`

Рассчитывает итоговую стоимость заказа БЕЗ его создания. Используется для отображения итоговой суммы перед кнопкой "Оплатить".

Поля тела запроса — те же, что и при создании заказа (см. раздел "Создать заказ").

Ответ: объект с полями расчёта:

| Поле | Тип | Описание |
|------|-----|----------|
| items | array | Список позиций с рассчитанными ценами |
| originalSubtotal | number | Сумма товаров без каких-либо скидок |
| productDiscountAmount | number | Сколько сэкономлено на скидках самих товаров |
| subtotal | number | Сумма после скидок на товары (до промокода и доставки) |
| discountAmount | number | Скидка от промокода (в рублях) |
| promoCode | string \| null | Применённый промокод (null если не использовался) |
| deliveryPrice | number | Стоимость доставки |
| total | number | Итоговая сумма к оплате (subtotal − discountAmount + deliveryPrice) |
| currency | string | Валюта (RUB) |

Поля каждой позиции в `items`: productId, productName, productSlug, size, color, quantity, price, discount, finalPrice, totalPrice, imageUrl (объект главного фото товара с полями url, order, isPreview, previewOrder).

## 3. Создать заказ

**POST** `/api/orders`

Создаёт заказ. Доступно гостям и авторизованным пользователям (у авторизованных заказ привязывается к аккаунту). Логика оплаты зависит от `paymentMethod`:

- `CARD_ONLINE` / `YANDEX_PAY` — заказ создаётся в статусе `PENDING` (`isPaid = false`), платёж регистрируется во внешней платёжной системе, в ответе возвращается `paymentUrl`. Фронт должен немедленно перенаправить пользователя на `paymentUrl`. После успешной оплаты статус станет `CONFIRMED` / `isPaid = true` (обновляется через callback платёжной системы или через проверку статуса, см. [payment.md](./payment.md)).
- `CARD_MANUAL` / `CRYPTO` / `PAYPAL` — заказ создаётся сразу в статусе `CONFIRMED` (`isPaid = true`), оплата обрабатывается вручную менеджером.

Email с подтверждением заказа отправляется в обоих случаях.

Формат номера заказа: `SALIYYYMMDDXXXXX`
- SALIY — префикс бренда
- YY — год (26 для 2026)
- MM — месяц (01-12)
- DD — день (01-31)
- XXXXX — уникальный 5-значный порядковый номер за день

Пример: `SALIY2603300001` (первый заказ за 30 марта 2026).

Важно:
- Цены берутся из БД, не от клиента.
- Проверяется наличие товара на складе.
- Уменьшаются остатки на складе (в том числе для `CARD_ONLINE` — до фактической оплаты, чтобы товар не увели).
- Работает для гостей и авторизованных пользователей.

### Поля тела запроса

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| items | array | Да | Список позиций (минимум 1) |
| firstName | string | Да | Имя |
| lastName | string | Да | Фамилия |
| email | string | Да | Email (валидируется) |
| phone | string | Да | Телефон |
| socialContact | string | Нет | Контакт в соц. сети (например, "Telegram: @ivan_petrov" или "Instagram: @ivan.p") |
| deliveryType | DeliveryType | Да | Тип доставки (CDEK_PICKUP / STANDARD) |
| paymentMethod | PaymentMethod | Да | Метод оплаты |
| countryName | string | Нет | Страна доставки |
| regionName | string | Нет | Регион |
| cityName | string | Нет | Город |
| cdekCityCode | number | Нет | Код города CDEK |
| street | string | Нет | Улица |
| apartment | string | Нет | Квартира |
| postalCode | string | Нет | Почтовый индекс |
| pickupPoint | string | Нет | Код ПВЗ CDEK |
| promoCode | string | Нет | Промокод |
| comment | string | Нет | Любое пожелание или уточнение к заказу |

Каждая позиция в `items`:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| productId | number | Да | ID товара |
| size | string | Да | Размер |
| quantity | number | Да | Количество (минимум 1) |

### DeliveryType (тип доставки)

| Значение | Описание |
|----------|----------|
| CDEK_PICKUP | Самовывоз из ПВЗ CDEK (только для России и Беларуси). Цена считается через CDEK API на основе города получателя и веса товаров; при сбое API — fallback на админскую настройку `delivery_price_cdek` (по умолчанию 500₽). |
| STANDARD | Почтовая доставка (800₽) — для всех остальных стран. |

Используйте эндпоинт `/api/orders/delivery-options` для получения доступных типов доставки в зависимости от страны.

### PaymentMethod (метод оплаты)

| Значение | Описание | Что делает backend |
|----------|----------|--------------------|
| CARD_ONLINE | Онлайн-оплата картой через Альфа-Банк (тест — `alfa.rbsuat.com`, прод — `pay.alfabank.ru`) | Заказ `PENDING` → регистрирует платёж в Альфе → возвращает `paymentUrl` |
| YANDEX_PAY | Онлайн-оплата через Яндекс Пэй (тест — `sandbox.pay.yandex.ru`, прод — `pay.yandex.ru`) | Заказ `PENDING` → регистрирует платёж в Яндексе → возвращает `paymentUrl` |
| CARD_MANUAL | Оплата картой через менеджера | `CONFIRMED` сразу, `isPaid=true` |
| CRYPTO | Криптовалюта | `CONFIRMED` сразу, `isPaid=true` |
| PAYPAL | PayPal | `CONFIRMED` сразу, `isPaid=true` |

Примечание: эндпоинт `/api/orders/delivery-options` в данный момент возвращает в `paymentMethods` только `CARD_ONLINE`, однако поле `paymentMethod` при создании заказа принимает любое из значений перечисления выше.

Подробнее про Альфа-Банк: [payment.md](./payment.md).

### Ответ (для онлайн-оплаты `CARD_ONLINE` / `YANDEX_PAY`)

Объект заказа со следующими полями:

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | UUID заказа |
| orderNumber | string | Номер заказа (формат SALIYYYMMDDXXXXX) |
| firstName, lastName, email, phone | string | Данные клиента |
| socialContact | string \| null | Контакт в соц. сети |
| comment | string \| null | Комментарий к заказу |
| deliveryType | DeliveryType | Тип доставки |
| paymentMethod | PaymentMethod | Метод оплаты |
| paymentId | string \| null | ID платежа в платёжной системе |
| originalSubtotal | number | Сумма до всех скидок |
| subtotal | number | Сумма после скидок на товары |
| deliveryTotal | number | Стоимость доставки |
| discountAmount | number | Скидка от промокода |
| total | number | Итоговая сумма |
| status | OrderStatus | Статус заказа (PENDING) |
| isPaid | boolean | Оплачен ли заказ (false) |
| currency | string | Валюта (RUB) |
| paymentUrl | string | URL платёжной формы банка/Яндекса |
| promoCode | object \| null | Применённый промокод (только `{ code }`) |
| items | array | Позиции заказа |

Для `CARD_ONLINE` / `YANDEX_PAY` клиент должен немедленно перенаправить пользователя на `paymentUrl`, чтобы попасть на платёжную форму.

### Ответ (для `CARD_MANUAL` / `CRYPTO` / `PAYPAL`)

Тот же формат, но `status = CONFIRMED`, `isPaid = true`, `paymentUrl = null`, `paymentId = null`.

Примечание: `promoCode` при создании возвращает только `{ code }`. Для полных данных промокода (type, value) используйте **GET** `/api/orders/:orderNumber`.

## 4. Получить заказ по номеру

**GET** `/api/orders/:orderNumber`

Возвращает детальную информацию о заказе по его номеру.

Параметр пути:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| orderNumber | string | Да | Номер заказа (например, SALIY2603290001) |

Ответ: полный объект заказа с теми же полями, что и при создании (id, orderNumber, данные клиента, socialContact, comment, deliveryType, paymentMethod, originalSubtotal, subtotal, deliveryTotal, discountAmount, total, status, isPaid, currency, createdAt), массивом `items` и объектом `promoCode`.

Поля товаров (items):

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | UUID позиции |
| productId | number | ID товара |
| name | string | Название товара (снэпшот на момент заказа) |
| slug | string | Slug для ссылки на страницу товара |
| color | string | Цвет товара |
| size | string | Размер заказанного товара |
| quantity | number | Количество |
| price | number | Оригинальная цена за единицу |
| discount | number | Скидка товара в процентах |
| finalPrice | number | Финальная цена за единицу (с учётом скидки товара) |
| totalPrice | number | Общая стоимость позиции (finalPrice × quantity) |
| imageUrl | object | Главное фото товара — primary, или ближайшее по порядку (url, order, isPreview, previewOrder) |

Поле `comment` — комментарий к заказу (null если не указан).

Промокод (`promoCode`) — информация о применённом промокоде (null если не использовался):

| Поле | Описание |
|------|----------|
| code | Код промокода |
| type | Тип (PERCENTAGE, FIXED, FREE_DELIVERY) |
| value | Значение скидки |

## 5. Получить мои заказы

**GET** `/api/orders`

Требуется авторизация.

Возвращает список всех заказов текущего пользователя (отсортированы по дате создания, новые первые). В список попадают как заказы, привязанные к аккаунту (по владельцу), так и гостевые заказы, оформленные на ту же почту, что у пользователя — поэтому заказы, сделанные до регистрации, тоже видны. Дополнительно при входе гостевые заказы на подтверждённую почту привязываются к пользователю в базе.

Ответ: массив объектов заказов. Каждый объект содержит полную информацию о заказе в том же формате, что и **GET** `/api/orders/:orderNumber`.

Что возвращается:
- Полная информация о каждом заказе.
- Все товары с фотографиями, slug, размерами, `finalPrice`, `totalPrice`.
- `originalSubtotal` — сумма до всех скидок.
- Информация о промокоде (code, type, value), если был использован.
- Детали доставки и оплаты.
- Отсортировано по дате (новые первые).

## Flow оформления заказа

1. Пользователь заполняет форму с данными: ФИО, email, телефон; контакт в соц. сети (опционально); адрес доставки; тип доставки; метод оплаты (на выбор): `CARD_ONLINE` (Альфа-Банк) / `YANDEX_PAY` (Яндекс Пэй) / `CARD_MANUAL` / `CRYPTO` / `PAYPAL`.
2. Вызывается **POST** `/api/orders/calculate` — показывается итоговая сумма, проверяется наличие товаров, рассчитывается доставка.
3. Пользователь нажимает "Оплатить".
4. Вызывается **POST** `/api/orders` — создаётся заказ, уменьшаются остатки, отправляется email с подтверждением. Для онлайн-оплаты платёж регистрируется в платёжной системе и возвращается `paymentUrl`.
5. Редирект на оплату (только онлайн-оплата): фронт перенаправляет пользователя на `paymentUrl`; пользователь вводит карту на форме банка, проходит 3DS; банк редиректит обратно на `{FRONTEND_URL}/order/{orderNumber}?payment=success|fail`; банк параллельно вызывает callback backend → backend обновляет статус заказа.
6. Страница `/order/:orderNumber` на фронте: читает `?payment=success|fail`; на всякий случай вызывает проверку статуса платежа (если callback не дошёл); показывает актуальный статус заказа.

Детали интеграции Альфы: [payment.md](./payment.md).

## 6. Валидация промокода

**POST** `/api/promo/validate`

Проверяет промокод перед применением в заказе (опционально).

Поля тела запроса:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| code | string | Да | Код промокода |
| orderAmount | number | Да | Сумма заказа |
| cartItems | array | Да | Позиции корзины (productId, quantity, price) |

Ответ (валидный промокод): объект с полями `isValid` (true), `discount` (размер скидки в рублях), `message` (описание скидки) и `promoCode` (объект с id, code, type, value).

Ответ (невалидный промокод): объект с полями `isValid` (false), `discount` (0) и `message` (причина).

Возможные причины невалидности:
- Промокод не найден
- Промокод не активен
- Срок действия истёк
- Достигнут лимит использований
- Промокод недоступен для аккаунта
- Минимальная сумма заказа не достигнута
- Промокоды не применяются к новинкам в корзине

---

## Работа с промокодами

### Применение промокода
1. Опционально: проверить промокод через **POST** `/api/promo/validate`.
2. Добавить поле `promoCode` в запрос создания/расчёта заказа.
3. Если промокод невалиден — будет получена ошибка 400 с описанием причины.

### Ограничения
- Промокоды не применяются к товарам с меткой "NEW" (кроме FREE_DELIVERY).
- Скидка рассчитывается на сервере (цены не от клиента).
- Автоматически записывается история использования.
- Работает для гостей и авторизованных пользователей.

## Коды ошибок

| Код | Описание |
|-----|----------|
| 200 | Успешно |
| 400 | Некорректные данные / недостаточно товара / невалидный промокод |
| 401 | Требуется авторизация |
| 404 | Заказ или товар не найден |
| 429 | Превышен лимит запросов |
| 500 | Ошибка сервера |
