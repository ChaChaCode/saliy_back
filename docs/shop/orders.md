# API заказов

> **Примечание:** Модуль заказов в разработке. Документация будет дополнена.

## Структура заказа

```typescript
{
  id: number;              // ID заказа
  userId: string;          // ID пользователя
  status: order_status;    // Статус заказа

  // Данные доставки
  deliveryType: string;    // Тип доставки (CDEK, самовывоз)
  deliveryPrice: number;   // Стоимость доставки
  deliveryAddress: string; // Адрес доставки

  // Контактные данные
  firstName: string;       // Имя
  lastName: string;        // Фамилия
  phone: string;           // Телефон

  // Финансы
  totalAmount: number;     // Общая сумма заказа

  // Даты
  createdAt: Date;         // Дата создания
  updatedAt: Date;         // Дата обновления

  // Связи
  items: OrderItem[];      // Товары в заказе
}
```

### Статусы заказа (order_status)
- `PENDING` - Ожидает оплаты
- `PAID` - Оплачен
- `PROCESSING` - В обработке
- `SHIPPED` - Отправлен
- `DELIVERED` - Доставлен
- `CANCELLED` - Отменён

### Структура OrderItem
```typescript
{
  id: number;
  orderId: number;
  productId: number;
  quantity: number;        // Количество
  size: string;            // Размер
  price: number;           // Цена на момент заказа
  discount: number;        // Скидка на момент заказа
}
```

## Планируемые эндпоинты

### Создать заказ
**POST** `/api/orders`

**Требуется авторизация:** Да

**Тело запроса:**
```json
{
  "items": [
    {
      "productId": 1,
      "size": "M",
      "quantity": 1
    }
  ],
  "deliveryType": "CDEK",
  "deliveryAddress": "ул. Пушкина, д. 5, кв. 12"
}
```

---

### Получить заказы пользователя
**GET** `/api/orders`

**Требуется авторизация:** Да

### Получить заказ по ID
**GET** `/api/orders/:id`

**Требуется авторизация:** Да

---

### Отменить заказ
**POST** `/api/orders/:id/cancel`

**Требуется авторизация:** Да

## Процесс оформления заказа

1. Пользователь добавляет товары в корзину (фронтенд)
2. Пользователь заполняет данные доставки
3. Система рассчитывает стоимость доставки через CDEK API
4. Пользователь создаёт заказ (POST `/api/orders`)
5. Система:
   - Проверяет наличие товаров
   - Резервирует товары
   - Создаёт заказ со статусом PENDING
6. Пользователь оплачивает заказ (интеграция с платёжной системой)
7. После оплаты статус меняется на PAID
8. Менеджер отправляет заказ → SHIPPED
9. Клиент получает заказ → DELIVERED
